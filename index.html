博客第一弹——近期总结

从开始学习前端也过了将近两个月的时间了，来总结一下前段时间干过的事情。

  • 熟悉html标签（静态百度首页）
  • html+css与页面布局以及bootstrap（仿途牛首页）
  • javascript & jQuery（chrome插件-bilibili每日排行）
  • hexo + github pages个人博客搭建

1、静态的百度首页

主要是熟悉html相关的标签，没有太复杂的地方和坑。当然那个时候的代码长得十分混乱只为实现样式而没有考虑写代码的规范问题。

2、仿途牛首页

html+css以及页面布局，运用到了bootstrap框架。我采用的是bootstrap中文网上的文档：Bootstrap中文网
以下是在写的过程中卡了比较久的地方：

1）导航条

作为一个完全的新手，导航条还是略微有些棘手的。下面是两个导航条的方法，在写这个页面的前后期分别用到了：
a) 利用bootstrap官方文档组件中的导航条，并添加自己需要的样式
b) 直接设置横排的列表来实现导航条功能，样式均为自定义，也避免了自定义样式与bootstrap默认样式的冲突问题

2）轮播条

在写这个页面时仅学习了html+css ，故轮播条直接采用了bootstrap 文档中的Carousel插件

3）信息展示框

信息展示框的样式是背景图片上覆盖有半透明的文字介绍。让文字介绍覆盖图片主要利用到css样式中的position属性。（特别需注意fixed,absolute,relative之间的初始定义位置的区别以及与父级元素位置之间的关系，否则排版出来就是凌乱）当初刚写的时候就是没有分清他们之间的关系，以至于往往添加了一个新内容排版就变得一团糟。
下图是自己关于position属性的一些笔记，笔记大致来源于慕课网的教程。

另关于页面布局这个站里的东西感觉还不错，适合在对css布局有些了解之后理顺思路过一遍：学习css布局

3、bilibili每日排行插件

获取B站每日各区排行前十作品信息的chrome插件。
下面就是在折腾插件的时候遇到的坑了

1）信息获取与分析

由于B站首页上各版块的排行版信息自身就是调用了一个json文件，且改json 文件的url已知。故可以直接利用postman分析其json文件并提取所需要的内容。

2）网络请求

这一部分考虑纯js实现网络请求。可以用原生的js 来实现但有些麻烦。这里我采用了 jQuery ajax 。具体参数参见jQuery官方文档 jQuery

3）跨域访问

跨域是个大坑。由于从B站首页获取的json文件并不与我们所构建的页面同源，且出于安全性问题浏览器有一个同源策略，会对不同源站点间的相互请求进行限制。这时我们写的网络请求就不可行了，浏览器就会报错。下图为chrome中的报错

其中报错信息中有着这样一句“Access-Control-Allow-Origin”。Access-Control-Allow-Origin 即跨域资源共享

在浏览器端我是采用了一个插件来解决这个问题Access-Control-Allow-Origin
在平时非调试状态下建议把跨域设置给关掉，否则有部分站点的访问会出现问题。

在插件中的解决方式要比浏览器页面方便一些：在manifest.json中添加“permissions”

详情见chrome开发
